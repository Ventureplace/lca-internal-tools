Design Rubric

A structured scoring skill for evaluating Figma designs across four dimensions. Run it in Claude Code to generate an interactive heatmap scorecard with gap severity ratings.

Four dimensions

01

Trust

Does the design make users feel safe, confident, and informed? Clear sourcing, honest tone, appropriate disclaimers, transparency about AI limitations.

02

Differentiation / Delight

Does it feel meaningfully different from generic tools? Brand personality, moments of warmth, non-generic patterns that reflect the host app.

03

Sound UX

Is it usable, accessible, and clear? Tap targets, contrast ratios, predictable navigation, loading states, error handling, empty states.

04

Conversion / Findability

Does it drive adoption and return visits? Entry point clarity, onboarding clarity, re-engagement hooks, frictionless flows.

Grading scale

Strong

Well-executed, no significant gaps

Functional but basic

Works, but generic or missing opportunities

Gap that needs a plan

Clear problem that needs design attention before launch

How to use

Running the skill in Claude Code

1

Open Claude Code and start a new session. The figma-design-grader skill should be available in your workspace.

2

With Figma MCP connected: paste your Figma URL directly — Claude will pull design context, layer structure, and token usage automatically.

3

Without MCP: export and upload a screenshot of your design. Claude grades visually and marks structural cells as unable to assess.

4

5

Ask for a prioritized action list sorted by gap severity, or drill into any specific primitive for deeper analysis.

# Figma Design Grader

Grades a design across four dimensions and outputs an interactive heatmap scorecard with gap severity ratings.

---

## Step 1: Detect Input Tier

Before grading, determine what's available:

**Tier 1 — Figma MCP connected + URL provided**
Use `get_design_context` (includes screenshot + specs) and `get_metadata` for layer structure.

**Tier 2 — No MCP, but Figma URL provided**
Ask: "I don't have Figma MCP access — can you export and upload a screenshot?" Then proceed as Tier 3.

**Tier 3 — Image uploaded only**
Grade purely from visual inspection. Note that structural/token grading is unavailable without MCP.

---

## Step 2: Pull the Design (Tier 1 only)

1. Call `get_design_context` with file key + node ID from the URL
2. Call `get_metadata` on the same node for layer structure and naming
3. Optionally call `get_variable_defs` to check token usage vs hardcoded values

URL format: `https://figma.com/design/:fileKey/:fileName?node-id=1-2`
→ fileKey = `:fileKey`, nodeId = `1:2`

---

## Step 3: Grade Against the Rubric

### The Four Dimensions

| Dimension | What It Measures |
|---|---|
| Trust | Safe, confident, informed? Clear sourcing, honest tone, no false urgency, medical disclaimers, AI transparency. |
| Differentiation / Delight | Meaningfully different from ChatGPT or WebMD? Brand personality, warmth, non-generic patterns. |
| Sound UX | Usable, accessible, clear? Tap targets, contrast, CTAs, navigation, loading/error/empty states. |
| Conversion / Findability | Drives adoption and return visits? Entry point clarity, onboarding, re-engagement hooks, frictionless flows. |

### Grading Scale

| Score | Color | Meaning |
|---|---|---|
| Strong | Green (#d4edda) | Well-executed, no significant gaps |
| Functional but basic | Yellow (#fff3cd) | Works, but generic or missing opportunities |
| Gap that needs a plan | Red (#f8d7da) | Clear problem needing design attention before launch |

### Gap Severity (per primitive)

- **Low** — Mostly green, minor polish needed
- **Moderate** — Mix of green/yellow, some gaps to address
- **High** — Multiple yellow/red cells, significant design debt
- **Critical** — Predominantly red, fundamental problems

---

## Step 4: Identify Primitives

Grade these by default (skip absent ones, add others if needed):

- Entry point
- Onboarding / Welcome
- Consent & permissions
- Chat input & interaction
- Streaming / loading states
- Response rendering
- Sources & verifiability
- Contextual chips / suggestions
- Navigation & care routing
- Chat history
- Red-flag escalation
- Abstention ("I don't know") states
- Medication guidance UI
- Triage UI

---

## Step 5: Output the Scorecard

Generate an **interactive HTML visualization** — not a text table.

**Chart structure:**
- Rows = Primitives
- Columns = Trust | Diff/Delight | Sound UX | Conversion
- + MVP notes column (brief observation per primitive)
- + Gap severity badge column

**Color coding:**
- Green: #d4edda / #155724 → "Strong"
- Yellow: #fff3cd / #856404 → "Functional but basic"
- Red: #f8d7da / #721c24 → "Gap that needs a plan"
- Gray: #e9ecef → "Unable to assess"

**Gap badge colors:** Low = green | Moderate = yellow | High = orange (#fd7e14) | Critical = red

**Required:** Hover tooltips (1–2 sentence rationale), summary bar, legend, responsive layout.

Output as a single self-contained HTML artifact with inline CSS/JS. No external dependencies except Google Fonts.

---

## Step 6: Follow-up

After the chart, offer:
1. "Want me to go deeper on any specific primitive?"
2. "I can generate a prioritized action list sorted by gap severity if useful."

---

## Notes for Tier 3 (image only)

- **Trust:** disclaimers visible, tone of copy, source attribution
- **Diff/Delight:** visual distinctiveness, brand personality
- **Sound UX:** layout clarity, tap targets, contrast, hierarchy
- **Conversion:** entry point clarity, CTA visibility, friction points

Mark cells "Unable to assess" (gray) where the screenshot lacks enough information.
